Suburb pages framework

Suburb page framework for Melbourne and Sydney

Use this structure when creating suburb pages so each page stays useful, distinct, and easy to maintain from manual listing data.

Required sections for each suburb page

  • Suburb overview with city context (no fabricated claims)
  • Listings snapshot pulled from data/listings.json
  • Category shortcuts relevant to that suburb
  • Booking-style shortcuts (SMS/email, online booking, etc.)
  • Claim-listing CTA for owners

Manual quality rules

  • Only include facts verified from business websites
  • Do not copy review text or external review summaries
  • No guarantees or medical-outcome claims
  • Use the data dictionary for consistent field values
